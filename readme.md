# 人工智能课程大作业项目说明

## 一、项目名称

基于 Bottom-Up-Agent 的自底向上智能体学习与演示系统

---

## 二、项目简介

本项目是人工智能课程的大作业，参考并学习开源项目 **Bottom-Up-Agent** 的设计思想，围绕“智能体如何在缺乏人工先验知识的情况下，通过观察环境、执行动作、积累经验并逐步形成技能”这一问题，完成一个可演示、可分析的智能体系统说明与实践项目。

与传统依赖人工预设流程的智能体不同，Bottom-Up-Agent 强调“自底向上”的能力形成方式：智能体从屏幕图像、鼠标点击、键盘输入等最基础的交互出发，在探索中逐步发现有效操作，并将其沉淀为可重复调用的技能。该仓库 README 将其概括为 Zero-prior Learning、Unified Codebase、Experience-Driven Evolution 和 Visualization，并给出了在 Slay the Spire、Civilization V 等环境中的演示。

---

## 三、项目背景与意义

近年来，大语言模型和多模态模型的发展推动了智能体系统的快速进步。许多现有 Agent 更偏向“自顶向下”的任务执行模式，即开发者先设计好目标、流程和工具调用方式，智能体再按流程完成任务。这类方法虽然在结构化任务中效果较好，但也存在对人工规则依赖强、泛化能力有限、面对复杂开放环境时适应性不足等问题。

Bottom-Up-Agent 提出了一种不同的思路：让智能体不依赖预定义目标、API 或游戏知识，而是从原始像素输入与模拟鼠标/键盘动作出发，通过经验不断发现、改进和共享技能。对于人工智能课程而言，这种设计非常适合作为“大作业”主题，因为它能综合体现视觉感知、语言模型推理、动作执行、技能学习和系统集成等多个 AI 核心问题。

---

## 四、项目目标

本项目的主要目标如下：

1. 学习并理解 Bottom-Up-Agent 的整体设计思路；
2. 梳理其“环境感知—动作执行—技能生成—技能演化”的系统流程；
3. 完成对开源项目核心模块和运行方式的分析；
4. 结合人工智能课程要求，形成一份结构完整、逻辑清晰、可用于展示汇报的课程大作业文档；
5. 展示自底向上智能体与传统流程型智能体的差异及特点。

---

## 五、项目的主要内容

本项目主要围绕 Bottom-Up-Agent 的框架展开，内容包括：

### 1. 智能体总体思想分析
分析其“从低级动作到高级技能”的学习机制，理解智能体为什么不需要一开始就拥有完整的任务流程。

### 2. 系统模块分析
从代码结构入手，分析项目中的主控模块、视觉模块、动作模块、记忆模块、搜索模块与可视化模块之间的关系。

### 3. 运行流程梳理
结合配置文件与主程序入口，梳理项目从读取配置、初始化 Agent、与环境交互、执行技能到保存日志和可视化展示的全过程。仓库中 `run.py` 会读取 YAML 配置文件，实例化 `BottomUpAgent`，随后以 “Play the game” 为任务启动运行；配置目录中则包含 `sts_explore_claude.yaml`、`c5_explore_claude.yaml`、`sts_explore_gpt4o.yaml` 等不同环境与模型组合的配置文件。

### 4. 课程化总结
将开源项目的前沿研究思想整理为适合课堂展示和课程答辩的内容，包括项目背景、技术路线、模块作用、分工安排和实验体会等。

## 六、项目特色

1. **强调自主学习**  
   智能体不是依靠固定流程工作，而是在环境中逐步学习有效技能。

2. **结合视觉、动作与语言推理**  
   项目同时涉及图像感知、鼠标键盘执行以及大语言模型推理，不是单一模型调用。README 中明确提到其从 raw pixel inputs 和 simulated mouse/keyboard actions 出发运行。

3. **具有可视化与可分析性**  
   项目支持通过可视化模块查看技能树和调用日志，便于展示智能体的执行过程。README 给出了 `python BottomUpAgent/visualization.py` 的可视化方式。

4. **适合人工智能课程大作业**  
   该项目兼具理论性、系统性和展示性，能较好体现人工智能课程中的综合实践能力。

## 九、代码的作用

结合仓库目录与文件命名，可以将各部分代码作用概括如下。仓库顶层包含 `run.py`、`BottomUpAgent/`、`base_model/`、`config/` 等部分；其中 `run.py` 负责读取配置并启动主流程，`config/` 下存放不同任务和不同模型的 YAML 配置，`base_model/` 下提供 `gpt.py`、`claude.py`、`UI_TARS.py` 等基础模型接口，而 `BottomUpAgent/` 目录则集中放置智能体主体逻辑。

### 1. `run.py`
作为项目入口文件，负责读取配置文件、创建 `BottomUpAgent` 实例，并启动智能体运行流程。它相当于整个系统的启动器。

### 2. `config/`
存放不同环境和不同模型对应的配置文件，例如：
- `sts_explore_claude.yaml`
- `sts_explore_gpt4o.yaml`
- `c5_explore_claude.yaml`

这些文件用于设置运行环境、分辨率、模型类型和探索模式等参数。

### 3. `base_model/`
该目录包含 `gpt.py`、`claude.py`、`UI_TARS.py` 等文件，主要用于封装不同底层模型接口，为智能体的决策、推理和对话生成提供基础支持。README 也说明项目当前支持 OpenAI 与 Claude API，并把 UI-TARS 作为可选基线接口。

### 4. `BottomUpAgent.py`
从命名和入口调用关系来看，这是智能体主体类所在文件，负责组织整个 Agent 的运行流程，是系统核心控制模块。`run.py` 中正是从这里导入 `BottomUpAgent` 并启动运行。

### 5. `Brain.py`
从命名上看，可理解为智能体的“决策/思考模块”，主要负责推理、规划和策略选择。

### 6. `Eye.py`
从命名上看，可理解为智能体的“视觉感知模块”，主要负责获取和处理屏幕图像信息。

### 7. `Hand.py`
从命名上看，可理解为智能体的“动作执行模块”，主要负责鼠标、键盘等交互操作。

### 8. `Detector.py`
从命名和 README 中的视觉模型配置可推测，该模块主要与界面元素检测、区域识别等视觉处理任务有关。README 还要求下载并配置 Segment Anything Model（SAM）权重，说明项目中确实存在视觉检测/分割相关能力。

### 9. `LongMemory.py`
从命名上看，主要用于长时记忆管理，负责保存经验、技能或历史状态，以支持后续复用。

### 10. `Mcts.py`
从命名上看，对应 Monte Carlo Tree Search（蒙特卡洛树搜索）模块，用于在候选行为中进行搜索和选择。

### 11. `Teacher.py`
从命名上看，可理解为对智能体进行引导、评价或反思辅助的模块。

### 12. `UnifiedOperation.py`
从命名上看，主要负责对不同操作进行统一封装，方便在多个环境下复用。

### 13. `visualizer.py`
用于结果可视化。README 中说明可通过可视化服务查看智能体的技能树与调用日志。

## 十、运行环境与运行方式

根据 README，项目当前实现主要支持 Windows，推荐使用 Python 3.10 和 Conda 创建环境；模型接口方面支持 OpenAI 与 Claude API，还需要配置 `.env` 文件。项目还要求下载 SAM 权重并放到 `weights/` 目录下，然后在目标游戏已经启动且处于主界面的前提下，通过如下方式运行：  
`python -m run --config_file "config/sts_explore_claude.yaml"`  
或者  
`python -m run --config_file "config/c5_explore_claude.yaml"`。

## 十一、项目总结

本项目以 Bottom-Up-Agent 为基础，围绕“自底向上智能体”的设计理念展开学习与分析。通过对项目背景、系统结构、运行流程、代码模块和课程意义的整理，可以看出该项目不仅体现了当前人工智能领域中 Agent 研究的发展趋势，也非常适合作为课程大作业进行展示。它将视觉感知、动作执行、大语言模型推理和经验积累结合起来，展示了智能体如何从简单操作逐步形成复杂能力的过程。
