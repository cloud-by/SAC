# 基于 Bottom-Up-Agent 的自底向上智能体学习项目

## 一、项目简介

本项目是人工智能课程大作业，基于开源项目 **Bottom-Up-Agent** 进行学习、复现与课程化改造，围绕“智能体如何在缺乏人工先验规则的情况下，通过与环境的持续交互自主学习技能”这一问题展开研究与实现。

传统智能体系统通常采用“自顶向下”的设计方式，即由开发者预先设定目标、流程、子任务和工具调用规则，智能体更多承担执行者的角色。而本项目所参考的 Bottom-Up-Agent 提出了一种“自底向上”的智能体学习范式：智能体不依赖预定义任务流程，而是从环境的视觉观察和基础操作出发，通过不断试错、反思与技能沉淀，逐步形成可复用的技能库，实现能力的持续演化。

因此，本项目不仅是一个智能体演示系统，也是一项关于 **经验驱动学习（Experience-Driven Learning）**、**技能进化（Skill Evolution）** 与 **自主探索式 Agent** 的课程实践。

---

## 二、项目背景与意义

随着大语言模型和视觉语言模型的发展，智能体（Agent）逐渐从单轮问答系统发展为能够感知环境、规划步骤并执行任务的复杂系统。当前大量智能体框架采用的是基于人工设计流程的任务驱动范式，这类方法虽然在结构化任务中表现良好，但在开放环境中往往存在以下问题：

1. 对人工先验依赖较强，需要预先定义任务目标、工具调用方式和执行流程；
2. 泛化能力有限，一旦环境发生变化，已有流程可能失效；
3. 智能体更擅长“按流程执行”，而不是“通过经验学习”。

为了探索更加接近人类学习方式的智能体机制，本项目选取 Bottom-Up-Agent 作为研究对象。该框架强调让智能体从零开始，仅通过图像观察、鼠标键盘操作以及环境反馈，逐步总结出有效行为模式，并沉淀为技能。该思想对于理解未来智能体系统的演化方向具有重要意义。

从课程实践角度看，本项目能够帮助学生系统理解以下人工智能关键问题：

- 智能体与传统模型的区别；
- 感知、决策、执行之间的关系；
- 大语言模型在复杂交互系统中的作用；
- 自主探索、经验积累与技能泛化的实现思路。

---

## 三、项目目标

本项目的总体目标是基于 Bottom-Up-Agent 框架，完成一个可运行、可展示、可分析的智能体学习演示系统，并结合课程要求对其方法进行梳理、实现与总结。

具体目标如下：

1. 搭建 Bottom-Up-Agent 的基础运行环境，理解其代码结构与运行流程；
2. 复现智能体在图形化环境中的基本交互过程；
3. 分析其“技能生成—技能调用—技能评估—技能优化”的核心机制；
4. 将项目整理为适合课程汇报与展示的大作业形式，包括系统说明、实验流程、运行截图与结果分析；
5. 在课程层面总结“自底向上智能体”相较传统工作流智能体的特点、优势与不足。

## 四、核心思想

本项目采用“自底向上”的智能体构建思想。与传统自顶向下方法不同，本系统不直接向智能体提供完整的任务分解流程，而是让其从最基本的交互动作出发，在环境中不断探索并积累经验。

其基本思想可概括为：

- **感知环境**：智能体通过屏幕截图获取当前状态；
- **执行原子动作**：通过鼠标点击、拖拽、键盘输入等低层动作与环境交互；
- **形成候选技能**：将多个原子动作组合成行为片段；
- **根据反馈筛选技能**：保留能够引起有效环境变化的行为；
- **构建技能库**：将有效行为存储下来，在后续相似场景中重复调用；
- **持续反思与优化**：通过语言模型对已有技能进行重写、修正或淘汰。

这意味着智能体不是依靠人工硬编码“应该做什么”，而是在交互中逐渐学会“哪些行为是有用的”。

## 五、系统总体架构

本项目系统可划分为以下几个核心模块：

### 1. 环境感知模块
负责获取当前界面的视觉信息，包括屏幕截图、界面元素识别等。系统主要依赖视觉输入理解当前状态，而不是调用游戏或软件的内部 API。

### 2. 动作执行模块
负责执行具体操作，包括鼠标点击、移动、拖拽以及键盘输入等。该模块是智能体与外部环境建立联系的基础。

### 3. 技能生成模块
将低级原子动作组合为更高层的行为片段，并在环境中尝试执行，判断其是否产生有效反馈。

### 4. 技能库管理模块
保存已经验证有效的技能，并支持后续检索、调用、更新和淘汰。

### 5. 技能评估与优化模块
根据执行前后的环境变化，对技能进行隐式评估。若技能长期无效，则对其进行裁剪或重写。

### 6. 决策与推理模块
借助大语言模型完成当前状态理解、候选技能选择、技能描述生成和策略反思，是系统的“大脑”部分。

### 7. 可视化模块
用于展示技能树、调用记录、执行状态等信息，增强系统的可解释性与演示效果。

## 六、关键技术路线

本项目涉及的关键技术主要包括以下几个方面：

### 1. 大语言模型驱动的智能体推理
利用大语言模型完成状态分析、技能语义描述、行为反思与候选动作筛选。语言模型不仅参与文本理解，还承担决策与优化的作用。

### 2. 基于视觉的环境理解
系统通过图像而非结构化数据理解环境状态，更接近真实用户与计算机界面的交互模式。

### 3. 原子动作组合与技能抽象
将简单操作组合为较复杂的行为序列，是从“低层控制”走向“高层技能”的关键步骤。

### 4. 隐式奖励机制
项目不依赖明确的人工奖励函数，而是通过界面变化、进度变化等间接信号判断某项技能是否有效。

### 5. 技能进化机制
通过“保留有效技能、淘汰低效技能、重写问题技能”的方式，使技能库不断演进。

### 6. Monte Carlo Tree Search（MCTS）辅助技能选择
在候选技能较多时，系统通过搜索与评估机制选择更有潜力的行为路径，从而提高探索效率。

## 八、项目特色与创新点

相较一般课程作业中的问答系统、分类模型或简单聊天机器人，本项目具有以下特点：

### 1. 更强调“学习过程”而不是“直接输出结果”
本项目关注的是智能体如何通过经验积累逐渐变强，而不只是一次性完成任务。

### 2. 不依赖人工预定义完整流程
系统不采用固定工作流，而是让智能体逐步发现有效行为，体现出一定的自主性。

### 3. 结合了视觉感知、动作执行与语言推理
项目不是单一模态的模型调用，而是一个包含“看—想—做—反思”的完整闭环。

### 4. 具有较强的研究性
项目与当前智能体研究前沿方向密切相关，体现了从传统任务型 AI 向开放环境 Agent 的发展趋势。

## 九、实验与演示设计

为了完成课程大作业展示，本项目可从以下角度进行实验与演示：

### 1. 基础运行演示
展示智能体在目标界面中的观察、操作和执行流程，体现其完整运行机制。

### 2. 技能生成过程展示
记录智能体从单步动作到组合技能的形成过程，展示其“自底向上”学习特点。

### 3. 技能库变化展示
对比运行前后技能数量、技能调用频率和淘汰情况，反映系统的演化过程。

### 4. 结果分析
从有效操作比例、执行成功率、任务推进程度等角度分析系统表现。

### 5. 可解释性展示
通过日志或可视化页面展示智能体为什么做出某些选择，增强项目说服力。

## 十、项目难点

本项目的实现难点主要包括：

1. **环境搭建复杂**：涉及依赖配置、模型接口、权重文件及图形环境适配；
2. **视觉理解不稳定**：仅依赖截图理解界面，容易受到分辨率、UI变化等影响；
3. **动作空间大**：在没有先验约束时，点击位置和操作方式组合极多，搜索空间很大；
4. **技能抽象困难**：如何将低层动作沉淀为可复用技能，是系统性能的关键；
5. **实验代价较高**：智能体运行时间长，对模型调用成本和调试耐心都有要求。

这些难点也正是该项目具有研究价值的地方。

---

## 十一、项目不足与改进方向

虽然 Bottom-Up-Agent 展示了较强的探索式学习能力，但作为课程项目或原型系统，仍存在一些局限：

1. 当前技能更多表现为“记录并复现”的动作序列，泛化能力仍有限；
2. 对视觉界面和分辨率较为敏感；
3. 对大语言模型能力和 API 稳定性依赖较强；
4. 训练与运行成本较高，不利于大规模实验；
5. 跨环境迁移能力仍有待增强。

未来可从以下方向继续改进：

- 加强技能参数化表示，提高技能复用性；
- 提升视觉元素识别与定位精度；
- 引入更稳定的状态表示方法；
- 尝试从游戏界面拓展到软件自动化场景；
- 研究多智能体共享技能库的协同机制。

## 十二、课程大作业总结

本项目围绕 Bottom-Up-Agent 展开，完成了一个面向智能体学习机制的课程实践。与传统以规则或固定流程为核心的系统不同，本项目强调让智能体在环境中自主探索、逐步积累经验并形成技能库，体现了人工智能从“任务执行”走向“经验学习”的发展趋势。

通过本次大作业，我不仅学习了智能体系统的整体架构，也加深了对大语言模型、视觉感知、技能抽象与自主决策等问题的理解。该项目具有较强的综合性和前沿性，能够较好体现人工智能课程中“模型 + 系统 + 应用 + 思考”的综合训练目标。
